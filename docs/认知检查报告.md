## 特别说明: K1 芯片 NPU 误区勘正

### 核心结论: K1 **没有** 独立 NPU 硬件

通过查询进迭时空 (SpacemiT) 官方文档和技术发布信息，确认:

> "我们之所以说 SpacemiT Key Stone K1 是 AI CPU 是因为其内部**没有集成 NPU 内核**，我们没有采用常见的 Arm CPU+NPU 的方式，而是通过 **RISC-V CPU 指令集的拓展**来实现的，即加入了 **16 条 AI 指令集**来实现了 AI 性能的提升。"

— 进迭时空品牌营销总监 段佳惠, 第四届滴水湖中国 RISC-V 产业论坛 (2024.08.19)

**K1 的 AI 算力架构**:
- 2.0 TOPS AI 算力来源于 RISC-V **自定义 AI 指令扩展**，融合在 CPU 核内部
- 属于 "AI CPU" 概念，而非传统的 "CPU + NPU" 分离架构
- AI 指令运行在 CPU Cluster 上（主要是 Cluster0 的 4 个核 + 512KB TCM）
- 不涉及独立 NPU 硬件单元（如华为昇腾、RK3588 NPU 等）

**K1 官方的 AI 推理方案**:
| 方案 | 说明 | 状态 |
|------|------|------|
| ONNX Runtime + SpacemiT EP | 利用 RVV 1.0 + IME 硬件加速，是**官方推荐路径** | 成熟 |
| TensorFlow Lite | 通过 RISC-V 向量指令加速 | 支持 |
| "Spacengine" SDK | 进迭时空的 AI 推理加速库（软件层），底层仍依赖 RISC-V AI 指令 | 当前项目引用但未启用 |

## K1 芯片 AI 推理正确架构图

```
应用层 (system_controller)
    |
    v
推理管线 (inference_pipeline)
    |
    +--> YOLOv8 / SCRFD / ArcFace ONNX 模型
    |        |
    |        v
    +--> ONNX Runtime (libonnxruntime.so)
             |
             +--> SpacemiT Execution Provider (RVV 1.0 + IME)
             |        |
             |        v
             +--> RISC-V X60 自定义 AI 指令 (16条)
                      |
                      v
                  Cluster0 (4核, 512KB TCM, 256-bit Vector)
```

**关键原则**:
1. K1 **没有独立 NPU**，AI 算力来自 CPU 内嵌的 RISC-V 自定义 AI 指令
2. ONNX Runtime + SpacemiT Execution Provider 是官方推荐的 AI 推理路径
3. 不要使用 "NPU" 一词描述 K1 的 AI 能力，使用 "RISC-V AI 指令加速" 或 "X60 AI CPU"
4. 已全面清理项目中的 NPU 不当引用，替换为正确的 AI 加速术语
