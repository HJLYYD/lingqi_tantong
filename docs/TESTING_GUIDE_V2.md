## 二、测试一: 追踪稳定性验证

### 2.1 目标
单一目标在连续30分钟追踪过程中ID保持率达到99%以上。

### 2.2 测试场景

**场景A: 单人直线行走**
```
设置: 空旷走廊，单人在摄像头前来回行走
时长: 30分钟
速度: 正常步行 (~1.2 m/s)
距离: 距摄像头2-10米范围
```

**场景B: 单人复杂运动**
```
设置: 室内场景，单人进行:
  - 正常行走 (5分钟)
  - 快速行走 (5分钟)
  - 慢走+转身 (5分钟)
  - 蹲下站起 (5分钟)
  - 之字形行走 (5分钟)
  - 坐下+站起 (5分钟)
总时长: 30分钟
```

### 2.3 运行命令

```bash
# 视频文件模式（推荐先测试）
./lingqi_tantong --config configs/test_config.yaml \
    --video test_single_person_30min.mp4 \
    --output output/test1_tracking_stability/

# 实时摄像头模式
./lingqi_tantong --config configs/test_config.yaml \
    --mode realtime \
    --camera /dev/video0
```

### 2.4 数据收集

日志中提取以下信息:
```bash
# 查看输出日志中的追踪统计
grep "Tracker" output/log.txt | grep -E "(id_switches|reidentifications|partial_matches)"

# 查看每帧的追踪ID
grep "Frame.*objects" output/log.txt | awk '{print $2, $NF}'
```

### 2.5 判定标准

| 指标 | 通过值 | 优秀值 | 测试方法 |
|------|--------|--------|---------|
| ID保持率 | ≥99% | ≥99.9% | 唯一track_id数 / 实际换人次数 ≤1 |
| ID交换次数 | ≤5次/30min | 0次/30min | 统计track_id变化次数 |
| 追踪连续性 | ≥95%帧有追踪 | ≥99%帧有追踪 | 有追踪的帧数 / 总帧数 |
| 平均追踪时长 | ≥25min | ≥29min | 同一ID的最长连续出现时间 |

**计算公式**:
```
ID保持率 = (1 - ID交换次数 / 应追踪人数) × 100%
对于单人场景: ID保持率 = (1 - (唯一ID数 - 1)) × 100%
```

### 2.6 通过条件
- 单人30分钟测试中，track_id固定不变 (仅1个ID)
- 无追踪中断超过1秒 (30帧)

## 三、测试二: 多人同时识别验证

### 3.1 目标
支持至少10人同时识别与追踪，准确率不低于95%。

### 3.2 测试场景

**场景C: 递增人数**
```
按以下顺序逐步增加人数，每组持续2分钟:
  1人 → 2人 → 3人 → 5人 → 7人 → 10人
所有人应在画面中可见（不重叠），以正常速度行走
```

**场景D: 密集人群**
```
10人同时在画面中，进行以下活动:
  - 所有人静止 (1分钟)
  - 交叉行走 (2分钟)  ← 关键测试
  - 随机走动 (2分钟)
```

### 3.3 运行命令

```bash
./lingqi_tantong --config configs/test_config.yaml \
    --video test_multi_person_10.mp4 \
    --output output/test2_multi_person/
```

### 3.4 数据收集

```bash
# 统计每帧追踪到的不同ID数
grep "Frame.*objects" output/log.txt | \
    awk -F'objects, ' '{print $1}' | awk '{print $NF}'

# 统计总ID数
grep "track_id" output/log.txt | sort -u | wc -l

# 检测新增追踪事件
grep "new_tracks" output/log.txt
```

### 3.5 判定标准

| 人数 | 期望追踪数 | 通过阈值 | 优秀值 |
|------|-----------|---------|--------|
| 1人 | 1 | 1 | 1 |
| 2人 | 2 | 2 | 2 |
| 3人 | 3 | 2-3 | 3 |
| 5人 | 5 | ≥4 | 5 |
| 7人 | 7 | ≥6 | 7 |
| 10人 | 10 | ≥9 | 10 |

准确率 = 正确追踪人数 / 实际人数 × 100%

### 3.6 通过条件
- 10人场景持续追踪准确率 ≥95%
- 交叉行走后ID无互换 (各人保持原ID)
- 新人进入场景后在 ≤0.5秒 (15帧) 内被检测

## 四、测试三: 抗遮挡性能验证

### 4.1 目标
在30%-70%遮挡情况下，上半身/侧半身识别准确率不低于85%。

### 4.2 测试场景

**场景E: 下半身遮挡** (上半身识别)
```
设置: 人物站在桌子/围栏后方
遮挡程度:
  - 30%遮挡 (小腿被挡): 行走1分钟
  - 50%遮挡 (大腿以下被挡): 行走1分钟
  - 70%遮挡 (腰部以下被挡): 行走1分钟
记录: 每个遮挡级别的追踪持续性
```

**场景F: 侧身行走** (侧半身识别)
```
设置: 人物在摄像头前侧身行走
测试:
  - 左侧身行走 (右侧不可见): 1分钟
  - 右侧身行走 (左侧不可见): 1分钟
  - 侧身进出遮挡物: 2分钟
```

**场景G: 间歇性遮挡**
```
设置: 人物走过柱子/门框等垂直遮挡物
测试:
  - 单次遮挡1-2秒: 重复20次
  - 连续遮挡3-5秒: 重复10次
记录: 遮挡后恢复时是否保持原ID
```

### 4.3 运行命令

```bash
./lingqi_tantong --config configs/test_config.yaml \
    --video test_occlusion_scenarios.mp4 \
    --output output/test3_occlusion/
```

### 4.4 数据收集

```bash
# 部分身体匹配统计
grep "partial_matches" output/log.txt

# 遮挡场景追踪状态
grep -E "(is_occluded|occlusion_score|partial_body)" output/log.txt

# 重识别事件
grep "reid" output/log.txt
```

### 4.5 判定标准

| 遮挡程度 | 场景 | 通过阈值 | 优秀值 |
|---------|------|---------|--------|
| 30% | 下半身遮挡 | ≥85%识别率 | ≥95% |
| 50% | 下半身遮挡 | ≥85%识别率 | ≥95% |
| 70% | 下半身遮挡 | ≥85%识别率 | ≥90% |
| 侧身 | 左侧/右侧行走 | ≥85%识别率 | ≥95% |
| 间歇性 | 1-2秒遮挡×20次 | ≥90% ID保持 | 100% |
| 间歇性 | 3-5秒遮挡×10次 | ≥80% ID保持 | ≥90% |

识别率 = 正确检测+追踪的帧数 / 总帧数 × 100%

### 4.6 通过条件
- 所有遮挡级别识别率 ≥85%
- 遮挡恢复后ID保持一致 (重识别成功)
- 无超过5秒的追踪丢失

## 五、测试四: 综合压力测试

### 5.1 目标
验证系统在复杂综合场景下的整体稳定性。

### 5.2 测试场景

**场景H: 综合场景**
```
设置: 室外/大型室内空间
时长: 10分钟
包含:
  - 多人交叉行走 (3-5人)
  - 部分遮挡 (树木/柱子)
  - 人物进出画面
  - 蹲下/站起
  - 快慢交替行走
```

### 5.3 判定标准

| 指标 | 通过值 |
|------|--------|
| 系统无崩溃 | 必须 |
| 平均FPS | ≥2 (K1板) |
| ID交换次数 | ≤3次/10min |
| 内存使用 | 稳定, 无持续增长 |
| 日志无ERROR | 必须 |

## 七、问题诊断指南

### 7.1 ID交换仍然频繁

```bash
# 检查外观特征是否在正常工作
grep "appearance_feature" output/log.txt | head -20

# 检查级联匹配是否生效
grep "cascade" output/log.txt

# 调整参数建议:
# 增大 appearance_weight (0.35 → 0.50)
# 降低 iou_threshold (0.30 → 0.25)
```

### 7.2 多人检测仍然不足

```bash
# 检查YOLO11n是否在TRACKING模式下定期运行
grep "Cascade\[TRACK\].*yolo=" output/log.txt

# 如果yolo值始终为0，检查:
# 1. CASCADE_SECONDARY_INTERVAL 是否设置正确
# 2. YOLO11n模型是否成功加载
```

### 7.3 遮挡场景识别率低

```bash
# 检查部分身体检测是否生效
grep "is_partial_body\|upper_body\|side_body" output/log.txt

# 检查关键点验证器阈值
# 降低 upper_body_min_keypoints (4 → 3)
# 降低 side_body_min_keypoints (3 → 2)
# 降低 occlusion_score_threshold (0.40 → 0.30)
```
