# SpacemiT K1 Platform Facts

This document records the correct technical facts about the SpacemiT K1 SoC and Muse Pi Pro development board, ensuring consistent understanding during project development.

## 1. SoC Specifications

| Item | Fact |
|------|------|
| Chip Name | SpacemiT Key Stone K1 |
| CPU Cores | 8× X60 RISC-V 64GCVB / RVA22 |
| Dual-Cluster Architecture | Cluster0: 4 cores + 512KB TCM + 2TOPS AI; Cluster1: 4 general-purpose cores |
| Clock Frequency | Up to 2.0 GHz (MUSE Pi Pro actual ~1.6 GHz) |
| AI Compute | 2.0 TOPS @ INT8 (CPU-integrated fusion, not independent NPU) |
| Vector Extension | RVV 1.0, 256-bit width |
| GPU | IMG BXE-2-32 @ 819MHz, OpenGL ES 3.2 / Vulkan 1.3 |
| VPU | H.265/H.264/VP8/VP9 codec, 4K@60fps decode / 4K@30fps encode |
| Memory | LPDDR4X, up to 16GB |
| Typical Power | 3.5W |

## 2. AI Acceleration Mechanism (Key Understanding)

### K1 Has No NPU

K1's AI compute comes from **IME (Intelligent Matrix Extension)**, a RISC-V custom matrix instruction extension integrated inside the X60 cores in Cluster0. This is a CPU-AI fusion architecture, not a traditional CPU+NPU heterogeneous architecture.

### Correct Terminology Reference

| Incorrect Term | Correct Term | Note |
|----------------|-------------|------|
| NPU | IME / AI Acceleration | K1 has no independent NPU hardware |
| npu_adapter | ai_accel_adapter | Adapter layer naming in code |
| HAS_SPACENGINE_NPU | HAS_SPACEMIT_EP | Compile macro naming |
| Spacengine NPU | SpacemiT EP (Execution Provider) | ONNX Runtime acceleration provider |

### ONNX Runtime Acceleration Path

```
ONNX Runtime
  ├── CPU EP (default, pure RVV vector acceleration)
  └── SpaceMITExecutionProvider (SpacemiT EP)
       └── Accelerates INT8 quantized model inference using IME instructions
```

- **SpacemiT EP** auto-detects `.q.onnx` quantized models (generated by xquant tool)
- Non-quantized models (FP32) automatically fall back to CPU EP
- EP registration is bridged through `spacemit_ort_bridge.cpp`

## 3. Model Quantization

### xquant Tool

- Developed based on PPQ v0.6.6+
- Input: FP32 ONNX model + calibration data
- Output: INT8 quantized ONNX model (`.q.onnx` suffix)
- Quantization format: QDQ (QuantizeLinear / DeQuantizeLinear)
- Accuracy loss: typically < 1%
- Performance improvement: tens of times

### Quantization Command

```bash
python -m xquant -c config.json -i model.onnx -o model.q.onnx
```

## 4. Bianbu OS

- Ubuntu-based RISC-V deeply optimized operating system
- Version: Bianbu 2.x (K1 long-term support) / Bianbu 3.x (latest)
- Kernel: Linux 6.6 LTS
- ONNX Runtime installed via APT, includes SpacemiT EP support
